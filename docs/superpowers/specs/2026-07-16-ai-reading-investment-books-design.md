# 设计文档：AI 精读投资书籍 → 个人投资知识库（读书引擎 + Notion 笔记管道）

- 日期：2026-07-16
- 状态：已通过用户设计评审，待 spec 复核
- 作者：与用户协作 brainstorm 产出

## 背景与目标

用户想建立自己的投资体系，但目前对金融和投资了解很少，希望用 AI 精读相关书籍，代替自己逐字阅读，并沉淀成可复用的知识资产。

完整愿景（分三个子系统，本 spec 只覆盖第一个）：
- **A) 知识库/笔记系统** —— AI 读书产出结构化、可检索的笔记，存入 Notion。
- **B) 个性化学习课程网站** —— 基于笔记素材重组成学习路径（本地小网站/App）。
- **C) 对话式投资导师** —— 基于读过的书回答用户提问。

三者共享同一地基：**"AI 读书 → 提炼结构化内容"引擎**。本 spec 只设计并交付这个地基 = **第一个子项目：读书引擎 + Notion 笔记管道**。B、C 复用其产出，各自另行设计。

### 一个重要提醒（记录在案）
投资亏损多源于"知道但做不到"（知行合一），这是 AI 替读学不来的。本系统定位是**加速知识获取与结构化**，不替代用户自己的判断与实践。因此笔记里专门保留"需自己判断"的标记，提醒用户哪些地方必须亲自消化。

## 需求确认（来自 brainstorm）

| 维度 | 用户选择 |
|---|---|
| 最终产出 | A+B+C 全都要（本 spec 只做 A 的地基） |
| 书的来源 | 有电子书文件（EPUB/PDF/TXT），AI 读原文 |
| 使用形态（终态） | B+C：本地网站 + 对话导师；笔记记到 Notion |
| 阅读深度 | 分章精读（逐章产出高质量笔记，再合成总览） |
| Notion 形态 | B+C：总览页面 + 可跨书检索的观点数据库 |
| 运行方式 | 命令行脚本 |
| 模型取向 | 质量优先（Opus 4.8） |

### 成本预期（已用 claude-api skill 核准报价）
- Opus 4.8：输入 $5/百万 token，输出 $25/百万 token。
- 一本 20 万字中文书 ≈ 输入 35–45 万 tokens、输出 5–9 万 tokens。
- **单本成本 ≈ $3.75（约 27 元人民币）**，耗时约 8 分钟。

## 范围

### 本 spec 包含
- 读书引擎：解析电子书 → 逐章精读 → 合成全书总览与投资原则。
- Notion 笔记管道：写入总览页面 + 观点数据库。
- 命令行运行、断点续跑、本地落地备份、成本护栏。

### 本 spec 不包含（后续子项目）
- 学习课程网站（子项目 B）。
- 对话式投资导师（子项目 C）。
- 本地大模型 / 其他模型（当前固定用 Opus 4.8，但精读器设计为模型可替换）。

## 整体架构

四个独立模块，各有单一职责、通过明确接口通信、可独立测试：

```
电子书文件夹 (books/*.epub|pdf|txt)
        │
        ▼
  ┌─────────────┐
  │ 1. 解析切章  │  提取纯文本，按章节切分
  └─────────────┘
        │  章节列表 [{title, text}]
        ▼
  ┌─────────────┐
  │ 2. 逐章精读  │  每章 → Opus 4.8 → 结构化笔记 (JSON)
  └─────────────┘
        │  章节笔记列表
        ▼
  ┌─────────────┐
  │ 3. 合成总览  │  全部章节笔记 → 全书总览 + 投资原则
  └─────────────┘
        │  总览 + 观点条目
        ▼
  ┌─────────────┐
  │ 4. Notion写入│  ① 总览页面  ② 观点数据库(可跨书检索)
  └─────────────┘
        │
        ▼
  本地 Markdown 备份 (output/书名.md)
```

### 模块职责与接口

1. **解析器 (parser)**
   - 输入：书文件路径。
   - 输出：`[{title: str, text: str}]`（章节列表）。
   - 只负责解析，不涉及 AI 或 Notion。

2. **精读器 (reader)**
   - 输入：单章文本（+ 固定标签表作为可选项提示）。
   - 输出：该章结构化笔记 JSON（见数据结构）。
   - 只与 Opus 4.8 交互；模型名与 prompt 可配置替换。

3. **合成器 (synthesizer)**
   - 输入：全部章节笔记。
   - 输出：全书总览（一句话总结、核心论点、提炼的投资原则、疑问/待验证点）+ 观点数据库条目列表。

4. **Notion 写入器 (notion_writer)**
   - 输入：总览 + 观点条目。
   - 输出：写入 Notion 的总览页面与观点数据库记录。
   - 只懂 Notion API，可单独重跑。

## 数据结构

### ① 总览页面（每本书一个 Notion 页面）
```
📖 书名（作者）
├─ 一句话总结
├─ 全书核心论点（3-5 条）
├─ 提炼的投资原则 / 可执行清单        ← "体系"素材
├─ 逐章笔记（按章节标题分节）
│   └─ 每章：核心观点 / 论据 / 可执行要点 / 金句
└─ 我的疑问 & 待验证点（AI 标记需用户自己判断处）
```

### ② 观点数据库（一条记录 = 一个核心观点，可跨书检索）

| 字段 | Notion 类型 | 说明 |
|---|---|---|
| 观点 | 标题 (title) | 一句话陈述该观点 |
| 来源书 | 选择/文本 | 书名 |
| 章节 | 文本 | 出处章节 |
| 主题标签 | 多选 (multi_select) | 从固定标签表中选取 |
| 论据摘要 | 文本 | AI 对支撑逻辑的简述 |
| 可执行度 | 选择 | 原则 / 可直接执行 / 需自己判断 |
| 原文金句 | 文本 | 忠于原书的引用 |

### 固定主题标签表（关键决策）
主题标签**不由 AI 自由生成**，而是从一套预定义固定表中选取，避免同义标签碎片化（"止损""停损""风险管理"各来一个）导致跨书聚合失效。

初始标签清单（用户可随时增删，存于 `config.py`）：
`估值` `风险控制` `仓位管理` `市场心理` `择时` `财报分析` `资产配置` `安全边际` `复利` `分散化` `护城河/竞争优势` `宏观周期` `交易成本/税费` `投资纪律`

若某观点无法归入现有标签，精读器在笔记中标记"建议新增标签：XXX"，供用户审阅后决定是否扩充固定表。

## 处理流程

### 正常运行
```
python read_book.py books/聪明的投资者.epub

[1/4] 解析…… 识别出 20 章，共 24 万字 ✓
[2/4] 逐章精读…… (Opus 4.8)
      第 1/20 章 ✓  …… 第 20/20 章 ✓
[3/4] 合成全书总览 & 提炼投资原则 ✓
[4/4] 写入 Notion…… 总览页面 ✓  观点数据库 +37 条 ✓
      本地备份 → output/聪明的投资者.md ✓
完成！耗时 ~8 分钟，花费 ~$3.8
```

### 错误处理设计

1. **断点续跑（核心）**：每章读完即把笔记存到 `output/.cache/书名/第N章.json`。中途失败重跑时，跳过已完成章节，从断点继续，不重复消耗 API。

2. **章节切分失败兜底**：结构混乱的 EPUB/PDF 切不出章节时，按固定字数硬切（如每 8000 字一段），保证流程可跑，并提示切分质量。

3. **API 限流/超时**：使用 SDK 自带重试，叠加脚本层指数退避，遇 429 自动等待重试。

4. **Notion 写入失败**：本地已存完整 JSON + Markdown 备份，Notion 写入是最后一步且可单独重跑（`python push_to_notion.py 书名`），永不因 Notion 问题丢失精读成果。

5. **成本护栏**：开跑前估算 token 与费用，超过阈值（默认 $10）先请用户确认。

**核心原则**：精读成果一旦产出立即落地本地，绝不因后续步骤失败而丢失。

## 技术选型

- **语言**：Python 3（Windows 10）。
- **AI**：Anthropic 官方 SDK `anthropic`，模型 `claude-opus-4-8`；每章笔记用 `output_config.format`（结构化输出）保证干净 JSON。
- **解析**：`ebooklib`（EPUB）、`pypdf` / `pdfplumber`（PDF）、内置读取（TXT）。
- **Notion**：官方 `notion-client`。
- **配置**：`.env` 存 `ANTHROPIC_API_KEY`、`NOTION_TOKEN`；`config.py` 存固定标签表、模型名、成本阈值、Notion 数据库/页面 ID。

### 用户需准备（实现阶段逐步带做）
1. Anthropic API key（按量付费）。
2. Notion 集成 token + 授权一个数据库/页面给该集成。
3. 安装 Python 3 与相关库。

## 测试策略

- **单元测试（离线，不烧 API）**：解析器（切章正确性）、Notion 写入器（字段映射正确性）。
- **精读器/合成器**：先用单章短文本跑通、人工验收笔记质量，再整本运行，避免整本跑完才发现 prompt 问题。
- **端到端**：先用一本用户熟悉的书完整跑一遍，对照原书检查笔记忠实度与标签合理性，调好 prompt 后再批量。

**先跑通一本 → 验收满意 → 再扩展**，避免一上来批量烧钱。

## 后续子项目衔接

- **B) 学习课程网站**：读取本引擎产出的 Notion 数据库 / 本地 Markdown，重组为学习路径。另行 brainstorm。
- **C) 对话式投资导师**：基于同一批笔记素材做检索问答。另行 brainstorm。

本引擎的产出（结构化笔记 + 可检索观点库 + 本地 Markdown 备份）即为 B、C 的共同素材源。
